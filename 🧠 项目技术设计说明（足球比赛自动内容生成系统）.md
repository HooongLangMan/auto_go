# 🧠 项目技术设计说明（足球比赛自动内容生成系统）

## 一、项目目标

构建一个基于多 Agent 的自动化内容生产系统，实现：

-  自动抓取每日足球比赛 
-  自动筛选高价值比赛（3场） 
-  自动补全结构化数据（API + 槽位机制） 
-  自动生成： 
  -  微信公众号长图文（800–1500字） 
  -  小红书短内容（200–500字） 
-  自动生成配图 
-  自动分发至各平台（通过 MCP Server） 

------

## 二、核心技术栈

-  多 Agent 编排：LangGraph 
-  数据校验：Pydantic 
-  数据库：PostgreSQL 
-  外部数据源：API-Football 
-  分发系统：MCP Server（已有） 
-  LLM：任意支持 function calling 的模型 

------

## 三、系统整体架构

```
[Crawler]
   ↓
[Selection Agent]
   ↓
[Enrichment SubGraph]
   ↓
[Content Agent]
   ↓
[Image Generator]
   ↓
[Distribution Agent]
```

------

## 四、Graph State 强约束设计（必须严格执行）

```
class GraphState(TypedDict):
    fixtures: list[dict]  # 原始比赛列表
    selected_match_ids: list[int]  # 仅ID
    match_data: dict[int, MatchInfo]  # 强结构化数据
    contents: dict  # 各平台内容
    publish_status: dict  # 发布状态
```

------

## 五、数据模型定义（强约束）

使用 Pydantic：

```
class MatchInfo(BaseModel):
    match_id: int
    league: str
    match_time: datetime

    home_team: str
    away_team: str

    home_rank: int | None
    away_rank: int | None

    home_recent_form: list[str]  # 最近5场 W/D/L
    away_recent_form: list[str]

    injuries: list[str] | None
    odds: dict | None

    must_fill: bool = True
```

## 六、Agent 设计（强职责分离）

------

### 1️⃣ Selection Agent（筛选）

**输入：**

```
fixtures[]
```

**输出（严格限制）：**

```
{
  "selected_match_ids": [1,2,3]
}
```

**约束：**

-  不允许输出额外字段 
-  不允许补数据 
-  只做“选择” 

------

### 2️⃣ Enrichment SubGraph（核心🔥）

#### Slot 定义：

```
REQUIRED_SLOTS = [
    "home_team",
    "away_team",
    "match_time",
    "home_recent_form",
    "away_recent_form"
]
```

------

### 子流程：

```
[检查槽位]
   ↓
[缺失 → 调 tool]
   ↓
[Pydantic 校验]
   ↓
[写入 state]
```

------

### Tool（必须封装，不允许 Agent 自由请求）

```
get_match_detail(match_id)
get_team_stats(team_id)
get_odds(match_id)
get_injuries(team_id)
```

------

### 强约束：

-  所有输出必须通过 Pydantic 校验 
-  校验失败 → 重试（最多2次） 
-  再失败 → 标记数据不完整 

------

## 七、Content Agent（内容生成核心）

------

### 输入：

```
MatchInfo
```

------

## 1️⃣ 微信公众号生成规则（强约束）

### 字数：

800–1500字

------

### 结构模板（必须遵守）：

```
【标题】
必须：情绪化 + 结论前置 + 冲突

【开头（判断）】
直接给出比赛判断（胜/平/负 + 倾向）

【中段（分析）】
1. 排名对比
2. 近期状态
3. 关键球员 / 伤停
4. 数据支撑（可引用赔率）

【结尾（印证）】
强化观点 + 给出预测结果
```

------

### 示例 Prompt（核心约束）

```
你是一个专业足球分析师，擅长写公众号爆款文章。

要求：
- 字数：800-1500字
- 风格：专业 + 情绪张力
- 开头必须直接给出结论
- 禁止胡编数据
- 所有分析必须基于提供数据

输出格式：
{
  "title": "...",
  "content": "..."
}
```

------

## 2️⃣ 小红书内容规则（强风格约束）

### 字数：

200–500字

------

### 风格：

-  情绪化 
-  带争议 
-  可引战（但不违规） 
-  多用反问/吐槽 

------

### 结构：

```
开头：结论 + 挑衅
中间：理由（简化版）
结尾：引导评论（制造对立）
```

------

### 示例 Prompt：

```
你是一个小红书足球博主，粉丝年轻，喜欢争论。

要求：
- 200-500字
- 语气偏主观
- 可以有“这场还用看？”、“懂球的都懂”
- 必须制造讨论点

输出：
{
  "title": "...",
  "content": "..."
}
```

## 十三、你这个系统真正的核心

不是 LangGraph
 不是 Multi-Agent

👉 是这三件事：

1.  **数据完整性（Pydantic + Slot）** 
2.  **内容风格控制（Prompt 强约束）** 
3.  **分发稳定性（幂等 + 重试）**
